# Windeye 命名体系

## 系统命名

| 中文名 | 英文名 | 说明 |
|--------|--------|------|
| 风眼 | Windeye | 整体系统名称 |

**命名由来**：
- 风眼是风暴的中心，平静而清晰
- 寓意：在信息风暴中保持清醒，洞察一切

---

## 组件命名

| 中文名 | 英文名 | 说明 |
|--------|--------|------|
| 风雷达 | Wind Radar | 追踪组件，感知风向变化 |
| 每日信号 | Daily Signal | 简报组件，从噪音中提取信号 |
| 洞察 | Insight | 提炼组件，挖掘价值 |
| 数据中心 | Insight Hub | 数据组件，存储与检索 |

---

## 数据类型命名

| 中文名 | 英文名 | type | 说明 |
|--------|--------|------|------|
| 关注点 | Focus | `focus` | 追踪主题，我关注什么 |
| 信号 | Signal | `signal` | 每日简报，发现的新信息 |
| 追踪 | Tracking | `tracking` | 追踪记录，趋势/前沿/问题/洞察/结论 |

---

## 命名原则

1. **中文优先**：面向中文用户，中文名称易理解
2. **隐喻恰当**：风/雷达/信号，都是感知相关的隐喻
3. **简洁明了**：每个名称都有明确含义
4. **一致性**：中英文对应，概念统一

---

## 使用规范

### 代码中使用

```go
// 数据类型常量
const (
    TypeFocus    = "focus"
    TypeSignal   = "signal"
    TypeTracking = "tracking"
)
```

### 文档中使用

- 系统名称：Windeye（风眼）
- 组件名称：Wind Radar（风雷达）
- 数据类型：Focus（关注点）

### 对话中使用

- "添加一个关注点"
- "今天的信号"
- "追踪这条信息"
