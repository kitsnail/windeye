## 三、设计哲学

### AI 时代的人机分工

```
传统模式：
人：发现问题 → 分析问题 → 解决问题 → 总结经验
（全程参与，精力分散）

AI 时代模式：
人：定义问题（方向、意图、价值判断）
AI：发现问题 → 分析问题 → 解决问题 → 挖掘价值
人：最终决策
（人聚焦核心，AI 承载执行）
```

### 为什么"定义问题"是人的核心价值

| 能力 | 人 | AI | 说明 |
|------|-----|-----|------|
| 判断什么重要 | ✅ | ❌ | 价值判断是主观的，只有人知道自己要什么 |
| 确定方向 | ✅ | ❌ | 方向选择涉及人生经验、直觉、偏好 |
| 发现新问题 | ⚠️ | ✅ | AI 可以从海量信息中发现模式、异常 |
| 解决问题 | ❌ | ✅ | AI 擅长执行、搜索、分析、生成 |
| 挖掘价值 | ❌ | ✅ | AI 可以从信息中提炼洞察 |

**人的稀缺资源**：注意力、精力、时间
**AI 的优势**：处理海量信息、不知疲倦、快速迭代

## 五、核心组件

### 5.1 Insight Hub（数据层）

**职责**：数据存储与检索

**技术栈**：Go + PostgreSQL + Vue.js

**API 端点**：
- `GET /api/v1/items` - 查询条目
- `POST /api/v1/items` - 创建条目
- `PUT /api/v1/items/:id` - 更新条目
- `DELETE /api/v1/items/:id` - 删除条目

**数据结构**：
```json
{
  "type": "focus|signal|tracking",
  "title": "标题",
  "content": "内容",
  "source_url": "来源 URL",
  "status": "active|paused|archived",
  "metadata": {}
}
```

### 5.2 Wind Radar（风雷达）

**职责**：记录"我关注什么"

**本质**：问题定义的载体

**使用方式**：
```
用户：添加关注 llm-d 项目
AI：创建 Focus { type: "focus", title: "llm-d 发展动态", source_url: "..." }
```

### 5.3 Daily Signal（每日信号）

**职责**：发现新信息

**工作流程**：
```
读取所有 active 的 Focus
     ↓
对每个 Focus：
├── 访问 source_url（优先 GitHub API）
├── AI 分析内容，判断是否有新发现
├── 有 → 生成 Signal
└── 无 → 跳过
     ↓
汇总生成 Daily Signal → 存入 Insight Hub
```

## 六、设计原则

| 原则 | 说明 |
|------|------|
| **人定义问题** | Focus 由人创建，表达"我关注什么" |
| **AI 发现问题** | Daily Signal 由 AI 生成，发现"有什么新动态" |
| **AI 挖掘价值** | Insight 由 AI 完成，回答"对我有什么影响" |
| **人做决策** | 最终判断由人完成，AI 只提供建议 |
| **数据极简** | 结构简单，AI 理解意图，自然扩展 |
| **持续迭代** | Focus 可调整，反馈闭环优化 |
