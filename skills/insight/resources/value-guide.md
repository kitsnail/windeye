# 价值评估指南

## 评估框架

Insight 的价值评估基于三个维度：

```
价值 = 决策影响 × 0.4 + 行动触发 × 0.3 + 认知改变 × 0.3
```

---

## 评估维度

### 1. 决策影响（40%）

**问题**：这条信息是否影响决策？

| 等级 | 标准 | 分数 |
|------|------|:----:|
| **高** | 直接影响关键决策 | 1.0 |
| **中** | 影响次要决策或提供决策依据 | 0.6 |
| **低** | 仅供参考，不影响决策 | 0.3 |
| **无** | 与决策无关 | 0 |

**评估问题**：
- 这是否改变了我对形势的判断？
- 这是否影响我的优先级排序？
- 这是否改变我的行动计划？

### 2. 行动触发（30%）

**问题**：这条信息是否触发行动？

| 等级 | 标准 | 分数 |
|------|------|:----:|
| **高** | 需要立即采取行动 | 1.0 |
| **中** | 需要近期行动或调研 | 0.6 |
| **低** | 需要长期关注 | 0.3 |
| **无** | 无需行动 | 0 |

**评估问题**：
- 这是否需要我做什么？
- 这是否创造了新机会？
- 这是否减少了某些风险？

### 3. 认知改变（30%）

**问题**：这条信息是否改变认知？

| 等级 | 标准 | 分数 |
|------|------|:----:|
| **高** | 颠覆性认知改变 | 1.0 |
| **中** | 修正或补充认知 | 0.6 |
| **低** | 验证已有认知 | 0.3 |
| **无** | 无认知价值 | 0 |

**评估问题**：
- 这是否揭示了新的模式？
- 这是否挑战了某个假设？
- 这是否简化了复杂情况？

---

## 价值等级

### 🔥 高价值

**标准**：总分 ≥ 0.7

**处理**：
- 重点追踪
- 定期回顾
- 链接到结论

**示例**：
> "选型逻辑从「谁更快」转向「谁更适合」"
> - 决策影响：高（直接影响选型决策）
> - 行动触发：高（需要调整选型方法）
> - 认知改变：高（颠覆性改变）
> - 总分：1.0 × 0.4 + 1.0 × 0.3 + 1.0 × 0.3 = **1.0** → 🔥

### ⭐ 中价值

**标准**：总分 0.4 - 0.7

**处理**：
- 持续关注
- 关联分析
- 观察是否升级

**示例**：
> "llm-d 新增 ROCm 支持"
> - 决策影响：中（如需 AMD 加速器则重要）
> - 行动触发：低（暂无 AMD 需求）
> - 认知改变：低（验证生态扩展）
> - 总分：0.6 × 0.4 + 0.3 × 0.3 + 0.3 × 0.3 = **0.42** → ⭐

### 📌 低价值

**标准**：总分 < 0.4

**处理**：
- 定期审核
- 决定保留或归档

**示例**：
> "vLLM 修复了若干 bug"
> - 决策影响：低（不影响选型）
> - 行动触发：低（常规更新）
> - 认知改变：无（常规维护）
> - 总分：0.3 × 0.4 + 0.3 × 0.3 + 0 × 0.3 = **0.21** → 📌

## 不同类型的价值评估

### Trend（趋势）

**重点**：方向性、持续性、影响范围

| 维度 | 问题 |
|------|------|
| 决策影响 | 这个趋势是否会改变我的技术选型？ |
| 行动触发 | 我是否需要提前布局？ |
| 认知改变 | 这个趋势是否改变了我的判断？ |

### Question（问题）

**重点**：紧迫性、复杂性、解决价值

| 维度 | 问题 |
|------|------|
| 决策影响 | 这个问题是否阻碍决策？ |
| 行动触发 | 是否需要立即调研或解决？ |
| 认知改变 | 解决这个问题是否能提升认知？ |

### Insight（洞察）

**重点**：独特性、可迁移性、颠覆性

| 维度 | 问题 |
|------|------|
| 决策影响 | 这个洞察是否改变了决策依据？ |
| 行动触发 | 是否需要调整行动方向？ |
| 认知改变 | 这个洞察是否颠覆了原有认知？ |

### Conclusion（结论）

**重点**：可执行性、预期收益、风险

| 维度 | 问题 |
|------|------|
| 决策影响 | 结论是否直接指导决策？ |
| 行动触发 | 行动步骤是否清晰可行？ |
| 认知改变 | 结论是否带来新的理解？ |

---

## 评估流程

```
新发现 → 初步评级（📌）
     ↓
深入分析 → 计算得分
     ↓
确定等级 → 🔥/⭐/📌
     ↓
定期回顾 → 升级/降级/归档
```

---

## 注意事项

1. **主观性**：价值评估是主观的，依赖用户的具体情况
2. **时效性**：价值会随时间变化，需要定期回顾
3. **关联性**：单个发现价值可能有限，但关联起来可能很有价值
4. **行动导向**：高价值不等于必须行动，需要综合考虑资源限制
